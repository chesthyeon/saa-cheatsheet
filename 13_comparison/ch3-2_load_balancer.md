# Chapter 3-2. 로드밸런서: ALB vs NLB vs GLB vs CLB

## 0. 한 줄 요약

🔑 **"어떤 계층(L4/L7)에서 어떤 트래픽을 분산하는가"가 로드밸런서 선택 기준이다 — HTTP/HTTPS → ALB, TCP/UDP → NLB, 보안 어플라이언스 → GLB, 레거시 → CLB**

---

## 1. 전체 비교 매트릭스

| 기준 | ALB | NLB | GLB | CLB (레거시) |
|------|-----|-----|-----|-------------|
| **OSI 계층** | L7 (HTTP/HTTPS) | L4 (TCP/UDP/TLS) | L3 Gateway + L4 | L4/L7 (제한적) |
| **프로토콜** | HTTP, HTTPS, gRPC, WebSocket | TCP, UDP, TLS | IP (모든 트래픽) | HTTP/HTTPS, TCP/SSL |
| **라우팅** | 경로/호스트/헤더/쿼리 기반 | 포트 기반 | 없음 (어플라이언스로 전달) | 없음 (단순 분산) |
| **성능** | 충분 (대부분 워크로드) | **초고성능** (수백만 RPS) | 인라인 처리 | 낮음 |
| **고정 IP** | ❌ (DNS만 제공) | ✅ **AZ당 고정 IP / Elastic IP** | ❌ | ❌ |
| **SSL 종료** | ✅ | ✅ (TLS) | ❌ | ✅ |
| **Lambda 타겟** | ✅ | ❌ | ❌ | ❌ |
| **IP 타겟** | ✅ | ✅ | ✅ | ❌ |
| **WebSocket** | ✅ (네이티브) | ✅ (TCP로) | ❌ | ❌ |
| **지연 시간** | 보통 | **극저지연** | 투명 전달 | 보통 |
| **Cross-Zone LB** | 기본 활성 | 기본 비활성 (AZ별 과금) | 기본 비활성 | 기본 활성 |
| **사용 사례** | 웹 앱, API, 마이크로서비스 | 게임, IoT, 고성능 TCP/UDP | 방화벽/IDS/IPS 어플라이언스 | 기존 EC2-Classic |

---

## 2. 핵심 판별 플로우차트

```
시작: "로드밸런서 선택"
  │
  ├─ HTTP/HTTPS 웹 트래픽? ─── YES ──→ ALB
  │   ├─ 경로/호스트 기반 라우팅? ──→ ALB (규칙 기반)
  │   ├─ Lambda 타겟? ──→ ALB
  │   └─ gRPC? ──→ ALB
  │
  ├─ TCP/UDP 고성능 + 고정 IP? ─── YES ──→ NLB
  │   ├─ Elastic IP 필요? ──→ NLB
  │   ├─ 극저지연? ──→ NLB
  │   └─ IP 화이트리스트? ──→ NLB
  │
  ├─ 보안 어플라이언스 (방화벽/IDS)? ─── YES ──→ GLB
  │
  └─ EC2-Classic / 레거시? ─── YES ──→ CLB (마이그레이션 권장)
```

---

## 3. 쌍별 상세 비교

### ALB vs NLB

| 기준 | ALB | NLB |
|------|-----|-----|
| **핵심 차이** | 콘텐츠 기반 라우팅 (L7) | 연결 기반 분산 (L4) |
| 라우팅 규칙 | 경로(`/api/*`), 호스트, 헤더, 쿼리 | 포트 기반만 |
| 고정 IP | ❌ DNS 이름만 | ✅ AZ당 1개 고정 IP (Elastic IP 연결 가능) |
| 성능 | 초당 수만~수십만 요청 | **초당 수백만 요청**, 극저지연 |
| Sticky Session | ✅ (쿠키 기반) | ✅ (소스 IP 기반) |
| 헬스 체크 | HTTP/HTTPS (상세) | TCP/HTTP/HTTPS |

**시험 빈출 판별:**

| 시험 키워드 | 정답 |
|------------|------|
| "경로 기반 라우팅", "호스트 기반 라우팅" | **ALB** |
| "마이크로서비스별 URL 라우팅" (`/users`, `/orders`) | **ALB** |
| "고정 IP", "Elastic IP", "IP 화이트리스트" | **NLB** |
| "극저지연", "수백만 연결" | **NLB** |
| "TCP/UDP 프로토콜" (비HTTP) | **NLB** |
| "게임 서버", "IoT 디바이스" | **NLB** |
| "Lambda 타겟" | **ALB** |
| "gRPC" | **ALB** |

### ALB vs CLB

| 기준 | ALB | CLB |
|------|-----|-----|
| 세대 | 현재 세대 | **레거시** (마이그레이션 권장) |
| 라우팅 | 경로/호스트/헤더 기반 | 없음 (단순 라운드 로빈) |
| 타겟 | 인스턴스, IP, Lambda | 인스턴스만 |
| 컨테이너 지원 | ✅ (동적 포트 매핑) | ❌ (고정 포트만) |
| 다중 포트 | ✅ (하나의 LB로 여러 서비스) | ❌ (서비스당 1개 CLB) |

> 💡 **시험 팁**: CLB가 정답인 경우는 거의 없음. "기존 CLB를 교체" → ALB 또는 NLB로 마이그레이션

### NLB + ALB 조합

```
클라이언트 → NLB (고정 IP) → ALB (L7 라우팅) → 타겟 그룹
```

| 사용 사례 | 설명 |
|----------|------|
| "고정 IP + HTTP 라우팅 모두 필요" | NLB 앞단 (IP 고정) → ALB 뒤단 (콘텐츠 라우팅) |
| "PrivateLink + HTTP 라우팅" | PrivateLink는 NLB 필요 → NLB 뒤에 ALB |

> 💡 **시험 팁**: "고정 IP" + "경로 기반 라우팅" 동시 요구 → NLB + ALB 조합

### GLB (Gateway Load Balancer)

| 기준 | GLB |
|------|-----|
| **목적** | 서드파티 보안 어플라이언스에 트래픽을 투명하게 전달 |
| **프로토콜** | GENEVE (포트 6081) |
| **사용 사례** | 방화벽, IDS/IPS, 딥 패킷 인스펙션 |
| **동작 방식** | VPC 트래픽 → GLB → 어플라이언스 → GLB → 원래 경로 |
| **시험 키워드** | "인라인 보안 검사", "서드파티 방화벽", "모든 트래픽 검사" |

> ⚠️ GLB는 SAA에서 빈출은 아니지만, "모든 트래픽을 서드파티 방화벽으로 검사" 패턴이 나오면 정답

## 4. 헷갈리는 포인트 / 함정

### 함정 1: "고정 IP" → ALB 선택 ❌

- ❌ ALB는 고정 IP를 제공하지 않음 (DNS 이름만)
- ✅ 고정 IP 필요 → **NLB** (Elastic IP 연결 가능)
- 💡 "IP 화이트리스트" 키워드도 NLB를 가리킴

### 함정 2: "로드밸런서" → 무조건 ALB ❌

- ❌ HTTP가 아닌 TCP/UDP 트래픽이면 ALB 부적합
- ✅ TCP/UDP → NLB, HTTP/HTTPS → ALB
- 💡 프로토콜을 먼저 확인하는 습관

### 함정 3: Cross-Zone 로드밸런싱 기본값 차이

- ✅ **ALB**: 기본 **활성** (무료)
- ✅ **NLB**: 기본 **비활성** (활성 시 AZ 간 데이터 전송 비용)
- ✅ **CLB**: 기본 **활성** (무료)
- 💡 "AZ 간 균등 분배" 문제에서 NLB는 별도 설정 필요

### 함정 4: ALB + Lambda vs API Gateway + Lambda

- ✅ **ALB + Lambda**: 기존 ALB 인프라에 Lambda 추가, 단순 HTTP 라우팅
- ✅ **API Gateway + Lambda**: API 관리 기능 (스로틀링, API 키, 캐싱, Swagger)
- 💡 "API 관리 기능" → API Gateway, "기존 ALB에 Lambda 추가" → ALB

### 함정 5: NLB는 보안 그룹이 선택적

- ✅ ALB: 보안 그룹 **필수**
- ✅ NLB: 보안 그룹 **선택적** (클라이언트 IP가 타겟에 보존됨)
- 💡 "클라이언트 IP 보존" → NLB (ALB는 X-Forwarded-For 헤더로 전달)

## 5. 시험 빈출 시나리오 + 정답

| 시나리오 | 정답 | 이유 |
|----------|------|------|
| 마이크로서비스별 URL 라우팅 (`/api/users`, `/api/orders`) | **ALB** | 경로 기반 라우팅 |
| 여러 도메인을 하나의 LB로 (`a.com`, `b.com`) | **ALB** | 호스트 기반 라우팅 |
| 게임 서버, 수백만 동시 TCP 연결 | **NLB** | 극저지연, TCP, 고성능 |
| 고정 IP + 방화벽 화이트리스트 | **NLB** | Elastic IP 지원 |
| 모든 VPC 트래픽을 서드파티 방화벽으로 | **GLB** | 인라인 보안 어플라이언스 |
| Lambda 함수를 HTTP 엔드포인트로 노출 | **ALB** | Lambda 타겟 지원 |
| 고정 IP + 경로 기반 라우팅 동시 필요 | **NLB + ALB** | NLB(IP) → ALB(라우팅) |
| PrivateLink로 서비스 노출 | **NLB** | PrivateLink는 NLB 필수 |
