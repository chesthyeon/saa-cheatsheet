# Chapter 3-1. 컴퓨팅: EC2 vs Lambda vs Fargate vs ECS vs EKS vs App Runner

## 0. 한 줄 요약

🔑 **"서버 제어가 필요한가 → EC2, 아니면 얼마나 관리를 위임할 것인가"로 갈린다 — 제어↑ = EC2, 위임↑ = Lambda/App Runner**

---

## 1. 전체 비교 매트릭스

| 기준 | EC2 | Lambda | ECS on EC2 | ECS on Fargate | EKS | App Runner |
|------|-----|--------|------------|---------------|-----|------------|
| **추상화 수준** | IaaS (OS 제어) | FaaS (함수 단위) | CaaS (컨테이너+인스턴스) | CaaS (서버리스 컨테이너) | CaaS (K8s) | PaaS (소스→배포) |
| **서버 관리** | 직접 (패치/스케일링) | 없음 | EC2 인스턴스 관리 | 없음 | 노드 그룹 관리 (또는 Fargate) | 없음 |
| **최대 실행 시간** | 무제한 | **15분** | 무제한 | 무제한 | 무제한 | 무제한 |
| **스케일링** | ASG (분 단위) | 요청 단위 (초 단위) | ASG + Service Auto Scaling | Service Auto Scaling | HPA/VPA/Cluster Autoscaler | 자동 (동시 요청 기반) |
| **콜드 스타트** | 없음 (항상 실행) | 있음 | 없음 | 있음 (경미) | 없음 | 있음 (경미) |
| **비용 모델** | 인스턴스 시간 | 요청 수 + 실행 시간 | EC2 비용 | vCPU + 메모리 시간 | 클러스터 + 노드 | vCPU + 메모리 (활성 시간) |
| **유휴 비용** | 있음 | **없음** | 있음 | **없음** (태스크 0이면) | 있음 (클러스터 비용) | **없음** (요청 0이면) |
| **적합 워크로드** | 장시간, GPU, BYOL, 상태 유지 | 이벤트 기반, 짧은 작업 | 컨테이너 + EC2 제어 필요 | 컨테이너 + 관리 최소화 | K8s 기존 워크로드 | 간단한 웹 앱/API |

## 2. 핵심 판별 플로우차트

```
시작: "컴퓨팅 서비스 선택"
  │
  ├─ OS/커널/GPU/BYOL 필요? ─── YES ──→ EC2
  │
  ├─ 15분 이내 이벤트 기반 작업? ─── YES ──→ Lambda
  │
  ├─ 컨테이너 워크로드?
  │   ├─ 기존 Kubernetes? ─── YES ──→ EKS
  │   ├─ 서버 관리 원함? ─── YES ──→ ECS on EC2
  │   ├─ 서버리스 원함? ─── YES ──→ ECS on Fargate
  │   └─ 가장 간단한 배포? ─── YES ──→ App Runner
  │
  └─ 대규모 배치 작업? ─── YES ──→ AWS Batch
```

## 3. 쌍별 상세 비교

### EC2 vs Lambda

| 기준 | EC2 | Lambda |
|------|-----|--------|
| 선택 시점 | 장시간 실행, OS 제어, GPU, 상태 유지 | 이벤트 기반, 짧은 실행, 서버리스 |
| 실행 시간 | 무제한 | **최대 15분** |
| 비용 패턴 | 항상 실행 = 유휴 비용 발생 | 실행할 때만 과금 |
| 스케일링 속도 | ASG (분 단위) | 즉시 (요청 단위) |
| **함정** | "서버리스" 키워드 → EC2 ❌ | "15분 이상 실행" → Lambda ❌ |

> 💡 **시험 키워드**: "이벤트 기반 + 짧은 작업 + 서버리스" → Lambda, "GPU/BYOL/장시간" → EC2

### Lambda vs Fargate

| 기준 | Lambda | Fargate |
|------|--------|---------|
| 단위 | 함수 (코드 조각) | 컨테이너 (Docker 이미지) |
| 실행 시간 | 최대 15분 | 무제한 |
| 메모리 | 최대 10 GB | 최대 120 GB |
| 사용 사례 | 이벤트 처리, API 백엔드, 간단한 작업 | 마이크로서비스, 장시간 컨테이너 |
| **함정** | "컨테이너 워크로드" → Lambda ❌ (가능하지만 Fargate가 적합) | "간단한 이벤트 처리" → Fargate는 과도 |

> 💡 **판별**: "함수 단위 짧은 작업" → Lambda, "컨테이너 + 서버리스" → Fargate

### ECS on EC2 vs ECS on Fargate

| 기준 | ECS on EC2 | ECS on Fargate |
|------|------------|---------------|
| 서버 관리 | EC2 인스턴스 직접 관리 (패치, AMI) | **관리 불필요** |
| 비용 | EC2 비용 (Spot/RI 가능 → 저렴 가능) | vCPU+메모리 시간 (EC2보다 단가 높음) |
| GPU | **지원** | 미지원 |
| 데이터 볼륨 | EBS 마운트 가능 | 임시 스토리지 (EFS 마운트 가능) |
| **함정** | "운영 오버헤드 최소화" → EC2 시작 유형 ❌ | "비용 최적화" → Fargate가 항상 저렴하진 않음 |

> 💡 **판별**: "운영 부담 최소화" → Fargate, "비용 최적화 + GPU" → EC2 시작 유형

### ECS vs EKS

| 기준 | ECS | EKS |
|------|-----|-----|
| 오케스트레이션 | AWS 네이티브 | **Kubernetes** |
| 학습 곡선 | 낮음 (AWS 친화적) | 높음 (K8s 지식 필요) |
| 이식성 | AWS 전용 | **멀티 클라우드 이식 가능** |
| 기존 K8s 워크로드 | 매니페스트 재작성 필요 | **그대로 사용** |
| 비용 | 추가 비용 없음 | 클러스터 $0.10/hr 추가 |
| **함정** | "Kubernetes" 키워드 → ECS ❌ | "AWS 네이티브 + 간단" → EKS는 과도 |

> 💡 **판별**: "Kubernetes" 명시 또는 "멀티 클라우드" → EKS, 그 외 → ECS가 대부분 정답

### App Runner vs ECS on Fargate

| 기준 | App Runner | ECS on Fargate |
|------|------------|---------------|
| 설정 복잡도 | **최소** (소스/이미지 → 바로 배포) | 중간 (Task Definition, Service, ALB 설정) |
| 제어 수준 | 낮음 (제한된 설정) | 높음 (네트워크, 볼륨, 서비스 디스커버리) |
| 빌드 | 자동 빌드 (GitHub 연동) | 별도 CI/CD 파이프라인 필요 |
| 사용 사례 | 간단한 웹 앱, API | 복잡한 마이크로서비스 |
| **함정** | "세밀한 네트워크 제어" → App Runner 부적합 | "가장 간단한 배포" → Fargate보다 App Runner |

> 💡 **판별**: "가장 간단한 컨테이너 배포" → App Runner, "세밀한 제어" → ECS on Fargate

## 4. 시험 빈출 시나리오 + 정답

| 시나리오 | 키워드 | 정답 | 오답 함정 |
|----------|--------|------|----------|
| 이미지 업로드 → 자동 처리 | 이벤트 기반, 짧은 작업 | **Lambda** | EC2 (과도) |
| 기존 K8s 앱 → AWS 마이그레이션 | Kubernetes, 최소 변경 | **EKS** | ECS (매니페스트 재작성) |
| 레거시 앱 컨테이너화, 운영 최소화 | 컨테이너, 운영 부담 최소 | **ECS on Fargate** | ECS on EC2 (관리 필요) |
| GPU 기반 ML 학습 | GPU, 고성능 | **EC2 (P/G 타입)** | Lambda (GPU 미지원) |
| 소스 코드에서 바로 웹 API 배포 | 가장 간단, 자동 빌드 | **App Runner** | ECS (설정 많음) |
| 피크 트래픽 10배, 비용 최적화 | 트래픽 변동, Spot | **EC2 ASG + Spot** | Lambda (장시간이면 부적합) |
| BYOL 라이선스, 물리 서버 제어 | 전용 호스트, 라이선스 | **EC2 Dedicated Host** | 모든 서버리스 (불가) |
| 30분 이상 데이터 처리 작업 | 장시간 + 컨테이너 | **ECS/Fargate** 또는 **Batch** | Lambda (15분 제한) |

## 5. 검증 필요 항목 ⚠️

- [ ] Lambda 최대 메모리 (10 GB), 최대 실행 시간 (15분) 변경 여부
- [ ] Fargate 최대 vCPU/메모리 조합 최신 값
- [ ] App Runner 최신 제한 사항 (최대 인스턴스 수, 메모리 등)
- [ ] EKS 클러스터 비용 ($0.10/hr) 변경 여부
- [ ] ECS on Fargate GPU 지원 추가 여부
